# Single Server Setup

One server handles everything.

![[Pasted image 20260405172442.png|637|640x379]]

1. User types a URL
2. DNS turns that URL into an IP address
3. Browser sends a request to that IP
4. Server sends back data (usually as JSON)

## Why Add a Database?

- Server memory is temporary — restart = data gone
- Database = permanent storage
- Separate machines → grow independently

![[Pasted image 20260405175859.png]]

## Which Database?

### Relational (SQL)
- Examples: MySQL, Postgres
- Data in **tables** (rows & columns)
- Can connect tables together (joins)
- **Default choice**

### Non-relational (NoSQL)
- Examples: MongoDB, DynamoDB, Neo4j
- Flexible — no strict table structure
- Pick this when:
	1. Data has no clear structure
	2. Massive amounts of data
	3. Speed matters more than structure
